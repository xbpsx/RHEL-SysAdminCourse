**procfs** (pseudo-filesystem) — это виртуальная файловая система в Linux, которая существует только в оперативной памяти и не хранится на диске.

**Для системного администратора это главный инструмент для диагностики и управления системой.**

**Кратко:**

1. **Что это?**  
    Виртуальные файлы, отображающие информацию о **процессах** (программах) и состоянии **ядра** в реальном времени.

2. **Где находится?**  
    Смонтирована в каталог `/proc`.

3. **Что внутри?**

    - Директории с числами (например, `/proc/1234`) — это информация о процессе с PID 1234 (его статус, потребляемая память, открытые файлы, переменные окружения и т.д.).

    - Файлы в корне `/proc` — это информация о всей системе:

        - `cpuinfo` — информация о процессоре.

        - `meminfo` — подробная информация об использовании оперативной памяти и свопа.

        - `mounts` — список смонтированных файловых систем.

        - `version` — версия ядра.

        - `sys/` — каталог для изменения параметров ядра "на лету".


**Зачем это админу?**

- **Мониторинг системы:** Быстро посмотреть использование памяти (`cat /proc/meminfo`), загрузку CPU, список процессов (`ps`, `top` работают через `/proc`).

- **Диагностика проблем:** Узнать, какие файлы открыл процесс, какие библиотеки использует, почему он "висит".

- **Настройка ядра:** Изменить сетевые параметры, настройки виртуальной памяти через файлы в `/proc/sys/` (например, `sysctl` использует именно его).

**Простой пример для админа:

```bash
# Посмотреть, сколько всего работает процессов
ls /proc | grep -E '^[0-9]+$' | wc -l

# Узнать, сколько свободной памяти
cat /proc/meminfo | grep MemFree

# Включить маршрутизацию пакетов (как роутер)
echo 1 > /proc/sys/net/ipv4/ip_forward
```

**Вывод:** `/proc` — это "окно" в ядро Linux. Для админа это незаменимый инструмент для понимания того, что происходит внутри системы.
