### **SHR** — Shared Memory

- **Что:** Часть RES, которая используется совместно с другими процессами.

- **Обычно:** Библиотеки (libc, libssl и др.).

- **Зачем админу:**

    - Понимать реальное уникальное потребление: `Уникальная память ≈ RES - SHR`

    - Если SHR большой — это хорошо, экономит RAM

### Память (VIRT/RES/SHR):

1. **Память кончилась?** Смотри на **RES**, а не VIRT

2. **Кто виноват?** Сортируй по %MEM (в top — нажми `Shift+M`)

3. **Есть ли экономия?** Большой SHR — хорошо, библиотеки в памяти один раз

4. **Утечка памяти?** Если RES растёт у одного процесса — проблема в нём
