### **VIRT** — Virtual Memory Size

- **Что:** Вся виртуальная память процесса (вся "виртуальная улица").

- **Что включает:** Код + данные + библиотеки + отображённые файлы + зарезервированная, но неиспользуемая память.

- **Может быть больше RAM!** Это нормально.

- **Зачем админу:** Оценить масштаб процесса. Резкий рост может указывать на проблему.

### Память (VIRT/RES/SHR):

1. **Память кончилась?** Смотри на **RES**, а не VIRT

2. **Кто виноват?** Сортируй по %MEM (в top — нажми `Shift+M`)

3. **Есть ли экономия?** Большой SHR — хорошо, библиотеки в памяти один раз

4. **Утечка памяти?** Если RES растёт у одного процесса — проблема в нём
