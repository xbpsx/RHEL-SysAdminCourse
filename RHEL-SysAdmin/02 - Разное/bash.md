**Shell** (оболочка) и **Bash** — это программы-посредники между пользователем и операционной системой.

### **Bash (Bourne-Again SHell)** - это КОНКРЕТНАЯ ПРОГРАММА

**Что это:** Самый популярный вариант shell, улучшенная версия оригинального Bourne shell.

**Почему стал стандартом:**

- ✅ Богатые возможности (история команд, автодополнение)

- ✅ Совместимость со скриптами sh

- ✅ Гибкая настройка

- ✅ По умолчанию в большинстве дистрибутивов Linux

### **Как это выглядит на практике:**

```bash
# Вы вводите в bash:
ls -l /home

# Bash:
# 1. Анализирует команду
# 2. Вызывает системные функции ядра
# 3. Получает список файлов
# 4. Форматирует и показывает вам результат
```

### **Почему это важно:**

- **Разные shell = разные возможности**

    - Bash: универсальный, много функций
    
    - Zsh: улучшенное автодополнение
    
    - Fish: удобный для новичков

- **Скрипты пишутся под конкретный shell**

```bash
# Скрипт для bash
#!/bin/bash
echo "Привет, мир!"
```

**Итог:** Shell — это "рабочее окружение" в командной строке, а Bash — самый популярный и мощный вариант этого окружения, ставший стандартом де-факто в Linux-мире.
