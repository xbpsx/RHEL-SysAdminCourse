### Термины: "Программа" vs "Команда"

В контексте терминала Linux эти понятия тесно переплетены, но между ними есть тонкое различие:

- **Программа (Утилита)** — это исполняемый файл, находящийся на диске (часто в каталогах `/bin`, `/usr/bin`, `/sbin` и т.д.). Например, `ls` — это программа, файл которой находится по пути `/bin/ls`.

- **Команда** — это строка, которую вы вводите в оболочке (shell), чтобы что-то сделать. Команда _чаще всего_ вызывает одну из этих программ.

**Ключевой момент:** Когда вы вводите в терминале имя программы (например, `ls`), оболочка находит соответствующий исполняемый файл и запускает его. Поэтому в повседневной речи говорят "введи команду `ls`", подразумевая и программу, и действие.

---
### Какие бывают типы "команд" (программ) в терминале?

Их можно классифицировать по нескольким признакам.

#### 1. По происхождению и месту в системе:

- **Встроенные команды оболочки (Shell Builtins):**

    - Это не отдельные программы, а часть самой оболочки (bash, zsh и др.).
    
    - Они выполняются очень быстро, так как не требуется запускать новый процесс.
    
    - **Примеры:** `cd` (сменить директорию), `echo`, `export`, `alias`, `pwd`.
    
    - **Как проверить?** Команда `type` покажет, является ли команда встроенной:

```bash
type cd
# вывод: cd is a shell builtin
```

**Внешние программы (Utilities):**

- Это те самые отдельные исполняемые файлы.

- **Примеры:** `ls`, `grep`, `cat`, `ssh`, `find`, `vim`.

- **Как проверить?** `type` также покажет путь к программе:

```bash
type ls
# вывод: ls is /usr/bin/ls
```

**Псевдонимы (Aliases):**

- Это созданные пользователем сокращения для длинных команд.

- **Пример:** Часто `ll` является псевдонимом для `ls -alF`.

- **Как проверить?** С помощью `type` или `alias`:

```bash
type ll
# вывод: ll is aliased to `ls -alF`
```

- **Функции (Functions):**

    - Пользовательские или системные функции, объединенные в скрипты. Более сложные, чем псевдонимы.

---
#### 2. По основному назначению (категории команд):

Это самая практичная классификация для начинающего сисадмина.

| Категория                     | Что делают                                                  | Примеры команд                                                          |
| ----------------------------- | ----------------------------------------------------------- | ----------------------------------------------------------------------- |
| **Навигация и файлы**         | Перемещение по файловой системе, работа с файлами и папками | `ls`, `cd`, `pwd`, `cp`, `mv`, `rm`, `mkdir`, `rmdir`, `find`, `locate` |
| **Просмотр и редактирование** | Просмотр и изменение содержимого файлов                     | `cat`, `less`, `more`, `head`, `tail`, `nano`, `vim`, `grep`            |
| **Права доступа**             | Управление разрешениями для файлов и пользователей          | `chmod`, `chown`, `chgrp`, `sudo`, `su`                                 |
| **Процессы**                  | Управление запущенными программами (процессами)             | `ps`, `top`, `htop`, `kill`, `pkill`, `bg`, `fg`                        |
| **Системная информация**      | Просмотр сведений о системе и ресурсах                      | `uname`, `whoami`, `df`, `du`, `free`, `uptime`                         |
| **Сеть**                      | Диагностика и работа с сетевыми подключениями               | `ping`, `ssh`, `scp`, `wget`, `curl`, `ip`, `netstat`                   |
| **Архивация**                 | Сжатие и распаковка файлов                                  | `tar`, `gzip`, `zip`, `unzip`                                           |
| **Установка ПО**              | Управление пакетами и программным обеспечением              | `apt` (Debian/Ubuntu), `yum`/`dnf` (RedHat/CentOS), `pacman` (Arch)     |

### Как жить с этим многообразием? Стратегия изучения

Вы абсолютно правы — знать все наизусть не нужно. Вот как строится работа:

1. **Ядро повседневных команд.** Вы быстро запомните 20-30 команд, которые используете каждый день (`ls`, `cd`, `cat`, `grep`, `ssh` и т.д.).

2. **Умение пользоваться справкой (`--help` и `man`).** Это **важнейший навык**.
    
    - Почти у любой команды есть встроенная справка по ключу `-h` или `--help`:

```bash
ls --help
```

Более подробная и полная справка — в **руководствах (man pages)**:

```bash
man ls
```
 - (Для выхода из `man` нажмите клавишу `q`).

1. **Знание, "какая команда за что отвечает".** Вам не обязательно помнить все опции `find`, но вы должны знать, что для поиска файлов используется именно `find`, а не `grep`. А `grep` используется для поиска текста _внутри_ файлов.
    
2. **Использование истории и автодополнения.**

    - Нажатие стрелки `Вверх` позволяет пролистать предыдущие команды.
    
    - Клавиша `Tab` автоматически дополняет имена команд, файлов и каталогов.


**Вывод:** "Команда" в терминале — это, по сути, действие, которое вы выполняете, запуская соответствующую программу. Не пытайтесь выучить все сразу. Начинайте с базового набора, активно используйте справку, и ваш активный словарь команд будет естественно расти вместе с решаемыми задачами. Заметки для сложных команд с множеством опций — это нормальная и профессиональная практика.
