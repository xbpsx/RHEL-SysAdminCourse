**Символическая ссылка (symlink/soft link)**

Символическая ссылка - это специальный файл, который содержит путь к другому файлу или директории. Это указатель на местоположение, а не на сами данные.

**Особенности символических ссылок:**

- Содержит только текстовый путь к целевому файлу

- Может ссылаться на файлы в других файловых системах

- Работает с директориями

- Может стать "битой" (broken) если целевой файл удален или перемещен

- Занимает небольшое место (размер пути)

---

**Сравнение: Жесткая vs Символическая ссылка**

|Характеристика|Жесткая ссылка|Символическая ссылка|
|---|---|---|
|**Что ссылается**|На inode файла|На путь к файлу|
|**Файловые системы**|Только в пределах одной|Может пересекать|
|**На директории**|Не работает|Работает|
|**При удалении оригинала**|Файл остается (пока есть ссылки)|Ссылка "ломается"|
|**Размер**|Минимальный|Размер пути|
|**Определение**|`ls -l` показывает нормальный файл|`ls -l` показывает `->`|

---

**Практическое использование:**

**Создание:**

```bash
ln -s /путь/к/оригиналу имя_ссылки
```

**Просмотр:**

```bash
ls -l                    # Покажет ссылки и их цели
file имя_ссылки          # Покажет "symbolic link to..."
readlink имя_ссылки      # Покажет куда ссылается
```

**Примеры:**

```bash
ln -s /var/log/logfile mylog    # Ссылка на лог-файл
ln -s ../config.conf config     # Относительная ссылка
```

---

**Преимущества символических ссылок:**

- Гибкость (межфайловые системы, директории)

- Простота понимания (видно куда ведет)

- Не ограничены количеством

**Недостатки:**

- Хрупкость (ломаются при перемещении цели)

- Немного медленнее доступ

- Рекурсия (может создавать циклы)

Символические ссылки широко используются для организации файлов, создания псевдонимов и управления версиями программ.
