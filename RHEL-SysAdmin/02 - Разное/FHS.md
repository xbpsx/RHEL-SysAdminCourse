**FHS (Filesystem Hierarchy Standard)** — это **стандарт иерархии файловой системы**, который определяет, для каких целей должны использоваться конкретные директории в UNIX-подобных ОС, таких как Linux.

Если в Windows каждый диск — это независимое "дерево" со своей буквой, то в Linux всё — от корневого раздела до флешки — является частью **одного большого дерева каталогов**, которое "растёт" из корня `/`.

### Основная идея FHS: Единое дерево с логической структурой

- **Корень `/`** — основа всего. С него начинается вся файловая система.

- **Подключение устройств** — Флешки, внешние диски и другие файловые системы не получают отдельную букву, а **"монтируются"** (подключаются) в какую-либо пустую папку внутри этого единого дерева (например, в `/mnt/usb`).

### Ключевые директории FHS и их назначение:

| Директория  | Назначение (простыми словами)                                                                                          |
| ----------- | ---------------------------------------------------------------------------------------------------------------------- |
| **`/bin`**  | Основные программы, которые нужны для загрузки и работы в однопользовательском режиме (например, `ls`, `cat`, `cp`).   |
| **`/etc`**  | **Конфигурационные файлы** всей системы. Здесь лежат настройки программ и служб.                                       |
| **`/home`** | **Домашние директории пользователей.** Здесь хранятся личные файлы, настройки, документы. Аналог `C:\Users` в Windows. |
| **`/var`**  | **Переменные данные.** То, что часто меняется: логи (`/var/log`), кеши, файлы баз данных.                              |
| **`/tmp`**  | **Временные файлы,** которые можно удалять после перезагрузки.                                                         |
| **`/usr`**  | **Ресурсы пользователя.** Вторичная иерархия для программ и библиотек, не критичных для загрузки.                      |
| **`/boot`** | Файлы, необходимые для **загрузчика** (ядро, initramfs).                                                               |
| **`/dev`**  | **Файлы устройств.** Сюда попадают диски, терминалы, USB-устройства (принцип "всё — файл").                            |
| **`/opt`**  | **Дополнительное ПО.** Сюда часто устанавливаются проприетарные или большие программы (Google Chrome, MATLAB).         |

### Почему этот подход лучше?

1. **Единообразие:** На любом дистрибутиве Linux (Ubuntu, CentOS, Debian) вы найдёте те же самые стандартные папки. Это упрощает администрирование и написание скриптов.

2. **Логичность:** Зная стандарт, вы сразу понимаете, где и что искать: настройки — в `/etc`, логи — в `/var/log`, свои файлы — в `/home/ваше_имя`.

3. **Гибкость:** Разные разделы жёсткого диска (например, отдельный диск для `/home`) можно легко "встроить" в общее дерево, просто смонтировав его в нужную папку. Пользователи и программы даже не заметят разницы.

**Вывод:** FHS — это "**план застройки**" для файловой системы Linux, который создаёт порядок, предсказуемость и логическую структуру для тысяч файлов, из которых состоит ОС.
