### **NI** — Nice Value

- **Что:** "Вежливость" процесса. Влияет на PR.

- **Диапазон:** от -20 (самый высокий приоритет) до +19 (самый низкий).

- **Зачем админу:** Настраивать приоритет CPU для задач:

```bash
nice -n 10 ./script.sh    # Запустить с низким приоритетом
renice 5 -p PID           # Изменить приоритет работающего процесса
```

### Приоритеты (PR/NI):

- **Системные/критичные процессы:** могут иметь высокий приоритет

- **Фоновые задачи (бэкапы, сбор логов):** должны иметь низкий приоритет (`nice +15`)

- **Real-time (rt):** **Опасно!** Только для действительно критичных задач (аудио, промышленные контроллеры)
